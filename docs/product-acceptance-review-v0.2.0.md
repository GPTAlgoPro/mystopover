# 龙腾中转礼遇产品验收 Review v0.2.0

## 结论

当前演示已统一到 `docs/stopover-prd-v1.0.md` 的主产品口径：以休息室为信任锚，串联行李全托管、标准化城市微游/酒店休息、返场缓冲和误机保障。PC Web、Mobile Web、静态五步流程和 LLM 对话流程都围绕同一组 PRD 规则与代表性案例展开。

## PRD 统一规则

| 规则 | 演示口径 |
| --- | --- |
| MVP 范围 | 6-48 小时中转旅客 |
| 轻享包 | 6-8h，机场内休息、行李寄存、快速安检 |
| 微游包 | 10-18h 白天中转，行李全托管、4h 固定城市路线、接送、误机保障 |
| 过夜包 | 12-36h 跨夜/家庭中转，酒店/钟点房、接送、行李直送 |
| 行李 SLA | 交包后 15 分钟内 RFID + 拍照登记，每件最高 ¥5000 保障 |
| 返场规则 | 起飞前 90 分钟启动行李返场，起飞前 60 分钟必须到达安检口 |
| 异常处理 | 我方服务导致误机时，自动进入改签、餐宿补偿，客服 5 分钟内介入 |
| 合规边界 | 第一版不做签证/入境服务，不做真实支付或真实供应商库存 |

## 演示入口一致性

| 入口 | 当前内容 | 与 PRD 的对应 |
| --- | --- | --- |
| PC Web 首页 | 新加坡樟宜 10h 微游主案例 | 对应探索型商旅、微游包、固定路线和行李返场规则 |
| Mobile Web 礼宾 | 默认使用同一套新加坡 10h 角色模板 | 和 PC 礼宾共享角色模板、套餐推荐与履约动作 |
| 静态预订流程 | 4h 边界、10h 微游、23h 过夜、35h 高端私享 | 覆盖不可订购、核心微游、家庭过夜和高端加购 |
| LLM 动态流程 | `/api/concierge` 使用 PRD 知识 + 本地规则兜底 | 在线模型和无 Key 兜底都按同一套餐阈值、SLA 和边界回答 |
| 履约状态机 | 订单、行李、微游/酒店、误机保障可手动推进 | 证明不是静态故事板，而是可演示的业务状态闭环 |

## 产品答辩建议

1. 先讲用户焦虑：时间、行李、体验、误机。
2. 再讲产品解法：休息室信任锚 + 行李全托管 + 标准路线 + 误机保障。
3. 演示顺序建议：打开 PC 首页主案例，切到 AI 礼宾自然语言下单，再进入静态五步流程展示 QR/RFID/追踪，最后用演示控制台触发误机保障。
4. 技术表达只讲三层：LLM 负责理解和解释，PRD 规则负责边界和价格，状态机负责履约可追踪。
5. 商业结果落到 PRD 指标：90 天套餐渗透率 8%、ARPL ¥450+、行李托管转化率 30%、首批 3-5 个 PoC 枢纽。

## 当前边界

- Demo 不接真实支付、航班动态、行李 IoT/RFID、酒店 PMS、eSIM 或城市游供应商库存。
- Demo 不提供签证/入境代办；签证不确定场景会降级到机场内权益或人工核验提示。
- 演示数据用于产品验收和 hackathon 答辩，不代表真实库存、真实价格或真实赔付承诺。
